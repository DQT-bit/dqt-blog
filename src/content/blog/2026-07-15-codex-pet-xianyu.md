最近在闲鱼搜索 Codex 桌宠，结果比我想象中热闹。

![闲鱼上的 Codex 桌宠相关商品](/codex-pet-xianyu-search.png)

页面上既有 10 元的指令模板、十几元的互动工具，也有 99 元到 200 元的桌宠定制服务。同一个需求，已经被拆成模板、素材包、代制作和成品交付几档生意。

看到这里，我脑子里先冒出来一句话：**能用上codex的人为什么还不会制作桌面宠物呢？**

来，我们直接把制作方法拆开。

## 先从工具开始：桌宠靠的是 hatch-pet skill

制作 Codex 桌宠，不只是输入一句神奇提示词。OpenAI 提供了一套开源的 `hatch-pet` skill，它负责生成角色基础形象、9 组动作、最终精灵图和配置文件。

官方 skill 地址是：

```text
https://github.com/openai/skills/tree/main/skills/.curated/hatch-pet
```

网上有些教程会让你直接输入 `/Hatch Pet`。这个入口可能随版本变化，稳妥的做法是先确认 `hatch-pet` skill 已经安装，再让 Codex 调用它。

## 第一步：安装 hatch-pet

先把 Codex 更新到最新版。打开一个新任务，输入：

```text
$skill-installer hatch-pet
```

如果按名称没有找到，就把官方地址一起发给 Codex：

```text
请使用 skill-installer 安装官方 hatch-pet skill：
https://github.com/openai/skills/tree/main/skills/.curated/hatch-pet

安装完成后告诉我实际安装路径，先不要生成桌宠。
```

安装结束后重启 Codex。重新打开一个任务，再输入：

```text
请检查 hatch-pet skill 是否已经可用，只告诉我技能名称和 SKILL.md 的实际路径，不要开始生成。
```

能返回 `hatch-pet` 和对应路径，才算准备完成。找不到 skill 就继续往下做，最后通常只会得到几张普通图片，生成不了 Codex 能识别的桌宠文件。

## 第二步：把素材准备好

如果要做自家的猫狗，准备一张清楚的正面或四分之三侧面照片。脸、耳朵、四肢和尾巴尽量完整，不要被手、衣服或家具挡住。

还可以补一两张侧面照片，用来说明毛色、花纹和尾巴长度。但不要一次上传十几张光线和角度完全不同的图，参考越乱，角色越容易变脸。

正式生成前，先写清楚四项信息：

1. 桌宠叫什么名字；
2. 想要什么风格；
3. 哪些外形特征必须保留；
4. 哪些元素绝对不要出现。

官方 skill 支持自动判断风格，也支持像素、毛绒、黏土、贴纸、扁平矢量、3D 玩具和绘画等风格。真实宠物第一次制作，建议选贴纸或扁平矢量，轮廓和花纹更容易保持稳定。

可以直接复制下面这个模板：

```text
请使用 hatch-pet skill，把我上传的照片制作成 Codex 桌宠。

名字：橘宝
风格：贴纸风格，轮廓清楚，颜色简洁
必须保留：左眼旁边的橘色花纹、白色前爪、短尾巴、绿色项圈
不要出现：文字、对话框、地面阴影、发光特效和额外配饰

请先生成基础形象并让我确认。基础形象没有确认前，不要继续批量生成动作。
```

把名字和特征换成自己的内容即可。

## 第三步：先确认基础形象

`hatch-pet` 不会直接拼最终成品。它会先生成一张基础形象，后面的所有动作都要以这张图为准。

这一步至少检查五个地方：

1. 脸型和五官位置是否接近原图；
2. 耳朵、尾巴和四肢数量是否正确；
3. 毛色和花纹有没有画反或漏掉；
4. 项圈、帽子等配饰是否前后一致；
5. 缩小后还能不能一眼认出角色。

不要只说“感觉不像”，要指出具体位置。例如：

```text
基础形象暂不通过。请只修改以下三处：
1. 补回左眼外侧的橘色花纹；
2. 两只前爪都改成白色；
3. 尾巴缩短三分之一。

脸型、眼睛、绿色项圈和贴纸风格保持不变。修改后再次给我确认，不要生成动作。
```

确认无误后，再输入：

```text
基础形象通过。把它设为后续所有动作的唯一视觉参考，锁定脸型、身体比例、花纹、配色和配饰，继续生成完整桌宠。
```

## 第四步：看懂它要生成的 9 种状态

一次完整制作最多会产生 1 张基础形象和 9 组动作。9 种状态分别是：

1. `idle`：空闲时轻微呼吸或眨眼；
2. `running-right`：桌宠向右移动；
3. `running-left`：桌宠向左移动；
4. `waving`：挥手或挥爪；
5. `jumping`：跳跃；
6. `failed`：任务失败；
7. `waiting`：等待你确认或输入；
8. `running`：Codex 正在执行任务；
9. `review`：Codex 正在检查结果。

这里最容易误会的是 `running`。它表示 Codex 正在工作，不是让宠物原地跑步。真正负责左右移动的是 `running-right` 和 `running-left`。

让 Codex 继续时，可以把动作要求一起说清楚：

```text
继续使用 hatch-pet 完成 9 种状态。

idle 只做轻微呼吸和眨眼；
waiting 用抬头等待的动作；
running 表现为低头专注工作，不要原地跑步；
review 用歪头认真检查的动作；
failed 用轻微沮丧表现，不要添加红叉、文字或漂浮符号。

请保持角色大小、脚底位置、花纹和配色一致，并在每个阶段报告进度。
```

这个过程需要生成多组图片，速度不会特别快。中途看到 Codex 在准备动作、拼接精灵图和运行检查脚本，不要提前结束任务。

## 第五步：不要只看成品，要看检查文件

正常完成后，skill 会生成一张总览图和多段动画预览。让 Codex 把这些文件打开：

```text
请打开最终的 qa/contact-sheet.png，并逐个检查 qa/previews 目录里的动画预览。

重点检查角色是否变脸、大小是否跳动、脚底位置是否上下漂移、左右方向是否正确、透明背景是否有白边。发现问题先列清单，不要直接宣布完成。
```

总览图里，九行动作应该是同一个角色。可以有姿势变化，但脸型、物种、身体比例、花纹、材质和配色不能变。

每一格最终会被放进大约 `192×208` 的显示单元，所以细小项链、复杂文字和碎花图案往往看不清。桌宠轮廓清楚，比堆很多装饰更重要。

遇到常见问题，可以直接这样返工。

角色在某个动作里变脸：

```text
只返工出现身份变化的动作行。以已经确认的基础形象为唯一参考，锁定脸型、眼睛位置、耳朵、尾巴、花纹和配色，不要重新设计其他已通过的动作。
```

动画播放时忽大忽小：

```text
检查每一帧的角色高度、脚底基线和中心位置。修复尺寸跳动和上下漂移，保持动作变化，但不要缩放角色主体。
```

透明背景出现白边：

```text
修复角色轮廓周围的白边、残留底色和杂点。保持边缘完整，不要损伤耳朵、毛发、尾巴和细小配饰。
```

左右移动方向不对：

```text
检查 running-right 和 running-left。右移动作必须面向右，左移动作必须面向左，同时保持花纹和配饰的位置关系正确。
```

## 第六步：确认文件已经安装

最终能被 Codex 识别的桌宠，至少需要两个文件：

```text
pet.json
spritesheet.webp
```

Windows 默认会放在类似下面的位置：

```text
C:\Users\你的用户名\.codex\pets\桌宠名称\
```

让 Codex 最后报告一次：

```text
请确认桌宠已经安装到 .codex/pets 目录，并告诉我 pet.json、spritesheet.webp、contact-sheet.png 和动画预览的完整路径。
```

然后重启 Codex，进入 `Settings → Appearance → Pets`，点击刷新并选择新桌宠。需要唤醒或收起桌宠时，可以在输入框使用：

```text
/pet
```

## 如果还想花钱定制，至少拿到这些东西

照着上面的流程做一遍，基本就知道闲鱼卖家在交付什么了。如果还是想省时间找人制作，下单前确认对方是否提供：

1. 最终的 `pet.json` 和 `spritesheet.webp`；
2. 九种状态的完整动作；
3. 总览检查图和动画预览；
4. 至少一次角色变脸或动作错误的修改；
5. 安装路径说明，以及换电脑后的重新安装方法。

只发一张角色图片，或者只帮你安装现成桌宠，都不能算完整的定制交付。

一套真正能运行的 Codex 桌宠，需要经过基础形象确认、九组动作生成、精灵图拼接、动画检查和本地安装。现在步骤已经拆开了，值不值得花 199 元，读者也能自己判断。
